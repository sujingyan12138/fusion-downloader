## 关键问题
1. 项目当前通过何种接口抽象平台下载、进度回调和 GUI 任务？
2. 本机与打包环境是否已有可用的 `ffmpeg`，以及是否允许引入成熟解析依赖？
3. Bilibili 未登录状态下该测试视频能获得的实际最高画质是多少，是否需要登录态才能突破清晰度限制？
4. 如何复用独立 Bilibili 浏览器登录态，让普通账号/大会员账号自动获得各自权限内的最高画质？
5. YouTube 测试链接在公开访问条件下实际暴露哪些格式，最高质量是否需要额外运行时依赖？
6. 如何彻底移除脆弱的 Google 登录流程，同时不影响 Deno/EJS、分段下载和真实最高质量策略？
7. 当前 YouTube 最高视频流的耗时主要来自解析、顺序 Range 请求、音频下载还是 FFmpeg 合并？
8. 如何让 GUI 的速度档位真正作用于 YouTube 大文件传输，并在 CDN 不支持并行 Range 时安全回退？
9. 如何在不混入 Obsidian 仓库文件的前提下，仅提交 downloader 的源码、测试、文档和项目状态？
10. TikTok 单作品当前返回的是合并流还是分离流，如何避开较低优先级的带水印候选并稳定选择最高质量？

## 已做决策
| 决策 | 理由 |
|------|------|
| 长期规则写入 `AGENTS.md`，任务状态写入三个规划文件 | 避免把短期实现细节固化为项目规则 |
| 项目专属结论不写入跨项目经验总结 | 遵守用户指定的记忆边界 |
| Bilibili 作为独立平台模块接入，GUI 仅显示“视频媒体” | 保持现有架构边界并控制功能范围 |
| 最高画质使用分离流时必须通过 FFmpeg 无重编码合并 | 避免为了单直链降质，也避免产生无声成品 |
| 使用 `yt-dlp` 2026.07.04 或更高版本作为 Bilibili 解析依赖 | 复用成熟且持续维护的站点适配，避免绑定易变私有接口 |
| 下载到任务临时目录，FFprobe 验证音视频后再移动为成品 | 失败不留下被误认作完成品的文件 |
| 最高质量排序固定为分辨率优先，其次帧率、码率；不写死 1080p | 普通账号、未登录和大会员应由源站返回的可用格式决定上限 |
| Bilibili 使用独立浏览器 profile，Cookie 只在本机内存与忽略目录中使用 | 支持会员清晰度且避免与其他平台登录态混用或泄露 |
| `yt-dlp` 登录态使用域名受限 CookieJar，不再放进全局 `http_headers` | 消除新版弃用警告，并由 Cookie 作用域约束发送域名 |
| YouTube 仅支持公开可访问内容，不提供登录入口或账号 Cookie | Google 拒绝自动化浏览器登录，且测试链接未登录已验证可下载 4K50；不为非必要登录增加脆弱流程 |
| YouTube 普通 HTTP 流保留 4 MiB 块并按速度档位使用 2/4/8 路并行 Range；失败时重新解析并回退 yt-dlp 稳定分段 | 当前实测瓶颈是单连接吞吐；并行提升明显，同时不能牺牲对不支持 Range、签名过期和瞬时 403 的容错 |
| YouTube 阶段追加到现有融合下载器主经历，不另建重复经历 | 遵守“一项目一主文件”，保留完整演化链并避免索引分裂 |
| TikTok 首阶段只支持公开单作品链接，并按 `res → fps → br` 选择最高候选 | 控制功能范围；当前测试链接的最高无水印标记候选已同时包含 H.265 视频和 AAC 音频 |

## 遇到的错误
| 错误 | 尝试次数 | 解决方案 |
|------|---------|---------|
| PowerShell/Windows 下 `rg ... *.spec` 把通配符当作非法路径 | 1 | 后续由 PowerShell 先解析文件列表，或使用 `rg -g '*.spec'` |
| Bilibili URL 正则把中文逗号后的文字纳入查询参数，导致 1 项测试失败 | 1 | 在中文分享文案标点处显式截断 URL，并增加回归测试 |
| 真实下载命令的外层时限误设为 10 秒，任务在解析/下载前期被终止 | 1 | 保留代码内网络超时，外层改为长任务时限并用分段等待获取进度 |
| 第二次真实下载在所选 AV1 CDN 流上等待超过 3 分钟且文件为 0 字节 | 1 | 停止该任务，分别探测同清晰度 AV1/AVC 小范围响应后调整质量排序或网络参数 |
| 清理遗留进程与临时目录的组合命令被安全策略拒绝 | 1 | 分开执行温和进程终止；临时目录待确认无锁后再按已验证绝对路径清理 |
| 10 MiB Range 分段被 Bilibili CDN 以 Windows 10054 重置，3 次重试后失败 | 1 | 不重复该分段大小；探测更小分段后选择稳定上限 |
| 规划文件阶段更新补丁因上下文变化而校验失败 | 2 | 分文件读取并小范围合并，未覆盖现有内容或影响源码 |
| `check-complete.ps1` 未识别中文阶段格式并报告 0/0 | 4 | 人工核对 9 个阶段均为 complete，且无 pending/in_progress；以测试与交付物证据完成审计 |
| 提交前 `git diff --cached --check` 发现 `tests/__init__.py` 末尾多余空行 | 1 | 删除多余空行，重新暂存后再次执行空白检查 |
| YouTube 测试链接元数据可解析，但最高视频流正式下载返回 `HTTP 403` | 1 | 重新解析验证普通 GET/Range 后采用 4 MiB HTTP 分段；完整 4K50 下载成功 |
| 用 `pyi-archive_viewer -l` 搜索 `downloaders.youtube` 未命中，归档检查提前停止 | 1 | 不重复假设模块以明文条目存在；先检查归档实际命名，再结合 PyInstaller TOC 验证模块与捆绑二进制 |
| PowerShell 双引号中的 `rg` 正则被转义破坏并报 `unclosed group` | 1 | 改用单引号字面正则或 `Select-String -SimpleMatch`，不重复原命令 |
| YouTube 32 MiB 单 Range 基准在当前 CDN 节点 45 秒无响应并超时 | 1 | 不重复大块单连接；重新解析签名 URL，改用小 Range、逐策略捕获失败并比较并行请求 |
| 最终 PyInstaller 在覆盖 `dist/融合下载器.exe` 时返回 `WinError 5` | 1 | 先只读核对占用该精确 EXE 的进程；只终止本轮验证遗留进程或在确认后重试，不清理其他用户进程/文件 |
| 最终规划审计发现 `progress.md` 阶段 10 仍标为 `in_progress` | 1 | 将阶段日志状态同步为 complete 后重新执行无活动项检查 |
| TikTok 真实文件已下载成功，但标题中的 🙂 导致 PowerShell GBK 日志抛出 `UnicodeEncodeError` 并误报任务失败 | 1 | 将日志输出包装为编码失败时按实际控制台编码替换不可表示字符后重试，媒体文件名本身保持原始 Unicode |
